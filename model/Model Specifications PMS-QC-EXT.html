This document specifies the PMS-QC_EXT layer in a concise, technical form. 5. Semantic anchor, extension governance, and base/EXT boundary

5.1 Semantic anchor

The workflow model recorded here is an optional organizational abstraction for toolchains and analysis. It is explicitly non-canonical and does not redefine PMS or PMS-QC composition logic.

PMS-QC_EXT includes an optional semantic anchor for workflow modelling. A QC workflow may be modeled as a typed directed acyclic graph (DAG), where nodes represent macros, gates, measurements or policy checks, edges represent causal, temporal or data dependencies, and scopes are determined by active □-frames. This is a toolchain-oriented aid, not a replacement for canonical PMS composition rules.

5.2 Extension governance

The role of this file is defined as an optional paper-specific refinement layer within the PMS-QC repository ecosystem.

Extensions may:

  • define paper-specific refinements of generic PMS-QC macros,
  • introduce paper-specific chain templates for analysis and annotation,
  • add optional structural policy refinements and bounded governance heuristics,
  • add organizational or toolchain-oriented helper metadata when explicitly marked as non-canonical.

Extensions should not, without explicit revision:

  • silently redefine PMS or PMS-QC base semantics,
  • silently promote EXT refinements into the generic base layer,
  • present paper-specific constructs as generic PMS-QC standards,
  • treat governance-oriented refinements as validated deployment architecture,
  • treat helper metadata as if it modified canonical PMS operator logic.

8.2 Pattern notes

qpe_exact

Matches QPE preparation, controlled ladder refinement, Fourier-domain reframing, alignment and final measurement. The QFT-alignment stage is semantically ordered as Σ(Α(Φ(S))). The ladder component remains paper-specific and EXT-only.

grover_fixed_point

Standard Grover remains a base-layer construction without Ψ. This template adds EXT-bound Ψ regulation within the iterated block to model fixed-point or overshoot-regulated variants.

attractor_alignment

Useful where reframing and alignment are structurally foregrounded. This pattern is non-canonical and should not be conflated with the generic base macro semantics of QFT_ALIGN.

9.2 Policy-by-policy notes

qpe_depth_guard

Parameter: t_max (integer)

Rule: Θ^t(Ω ∘ □) requires t <= t_max unless an explicit Ψ-stabilisation section follows.

This is a paper-specific governance or audit bound, not a correctness or optimality guarantee.

grover_no_overshoot

Rule: If Σ is applied after Α in Grover-style cycles, Ψ should follow within the same iteration scope unless alternative paper-specific invariance conditions are explicitly declared.

basis_reframe_guard

Rule: After Σ, Φ may reframe only if accompanied by explicit Ψ invariants.

attractor_cycle_integrity

Rule: Α steps repeated under Θ should incorporate Σ checkpoints or Ψ invariants in order to avoid structurally unbounded divergence.

Policy interpretation rule

EXT policies are optional, paper-specific structural guardrails. They do not replace the base policy layer and do not validate safety, convergence, physical stability, or deployment readiness.
