EXT Examples

The following example suite gives YAML-compatible EXT-layer example blocks for pms-qc-ext.yaml. All examples are intentionally bounded and should be read as paper-specific structural renderings only. They do not imply physical correctness, convergence guarantees, implementation adequacy, safety, or deployment readiness.

These examples belong to the optional refinement layer. They must not be read as generic PMS-QC base examples, and none of them promotes paper-specific ladder logic, centering logic, attractor schedule templates, or measurement-domain refinements into the base overlay.

qpe_ladder_refined

Description. EXT-only QPE example with explicit reusable ladder refinement. This example makes the power-of-two controlled-unitary ladder visible as a paper-specific macro and therefore must not be read as a generic PMS-QC base rendering. 

QFRAME(eigenproblem)
QPREP(superposition)
QPE_ORACLE_LADDER(U, t)
QFT_ALIGN(inverse)
Σ
QMEASURE(basis=computational, register=control)
PMS expansion

Λ ∘ Δ ∘ Σ ∘ Α ∘ Φ ∘ Θ^t( Ω ∘ □ ) ∘ Α ∘ ∇ ∘ Δ

Interpretation. Structural rendering only. □: eigenproblem frame; Δ/∇/Α: preparation of the control superposition; Θ^t(Ω ∘ □): explicit EXT-level reusable ladder refinement; Φ/Α/Σ: reframing, alignment and explicit structural commit for measurement, semantically read as Σ(Α(Φ(S))); Δ/Λ: phase-encoded alternatives resolved into a discrete estimate. This example is EXT-only because the named ladder macro is paper-specific and not part of PMS-QC base semantics.

EXT context

EXT macros: QPE_ORACLE_LADDER

Chain patterns: qpe_exact

Active EXT policies: qpe_depth_guard, basis_reframe_guard

grover_centered_refined

Description. EXT-only Grover example with paper-specific centering refinement. This example adds Ψ-bound regulation after attractor integration and must not be conflated with the standard base-layer Grover rendering. 

QFRAME(problem)
QPREP(superposition)
QITERATE(k, { QORACLE_CALL(F); GROVER_CENTERING; })
QMEASURE(basis=computational, register=search)
PMS expansion

Λ ∘ Δ ∘ Θ^k( Ψ ∘ Σ ∘ Α ∘ Ω ∘ Δ ∘ □ ) ∘ Α ∘ ∇ ∘ Δ ∘ □

Interpretation. Structural rendering only. □: problem frame; Δ/∇/Α: prepared search field; Ω: oracle asymmetry; Α/Σ/Ψ: paper-specific regulated amplification cycle introduced through GROVER_CENTERING; Θ^k: repeated bounded centering cycle; Δ/Λ: outcome distinction and unrealised branches. This is not standard Grover and therefore remains EXT-only.

Active EXT policies: grover_no_overshoot

attractor_fixed_point_linear

Description. EXT-only attractor-schedule example using a linear fixed-point-oriented refinement template. This example is intended for paper-aligned structural analysis of bounded attractor schedules and does not imply actual convergence or physical adequacy. 

QFRAME(problem)
QPREP(superposition)
QORACLE_CALL(F)
ATTRACTOR_FIXED_POINT(linear)
QMEASURE(basis=computational, register=search)
PMS expansion

Operators: Α, Θ, Σ, Ψ

Interpretation. Structural rendering only. ATTRACTOR_FIXED_POINT introduces a paper-specific attractor schedule template with attractor formation (Α), iterative contraction or repetition structure (Θ), checkpointed integration (Σ), and bounded invariance (Ψ). The EXT schema defines this construct as a refinement template rather than as a generic base macro with fixed canonical chain semantics. It therefore remains EXT-only and non-guarantee by design.

EXT context

EXT macros: ATTRACTOR_FIXED_POINT

Active EXT policies: attractor_cycle_integrity

PMS expansion

Operators: Δ, Λ, □

Interpretation. Structural rendering only. MEASUREMENT_PROJECTION_MODEL refines the generic measurement rendering by making outcome distinction (Δ), unrealised branches (Λ), and the relevant measurement domain (□) jointly explicit. The EXT schema defines this construct as a measurement-domain modelling refinement rather than as a replacement for QMEASURE. It therefore remains EXT-only and non-canonical.

attractor_alignment_refined

Description. EXT-only paper-specific reframing-plus-alignment example for spectral or Fourier-oriented estimation settings. This example foregrounds the attractor_alignment chain pattern and should not be conflated with the generic base semantics of QFT_ALIGN. 

QFRAME(eigenproblem)
QPREP(superposition)
QFT_ALIGN(inverse)
Σ
PMS expansion

Σ ∘ Α ∘ Φ ∘ Δ

Interpretation. Structural rendering only. Δ marks the relevant phase-distinguished alternative structure; Φ reframes from spectral or phase encoding toward readout space; Α aligns amplitudes in the reframed domain; Σ commits the aligned structure. This pattern is useful for paper-specific analysis, but it is non-canonical and should not be read as redefining the generic base macro language.

Boundary note

All five examples are EXT-layer examples. They presuppose optional paper-specific refinement logic and therefore must not be treated as generic PMS-QC base examples. In particular, reusable ladder constructs, centered Grover regulation, fixed-point attractor templates, and projection-domain refinements remain EXT-only.

Non-guarantee note

EXT usage does not imply physical correctness, stability, convergence, safety, or deployment adequacy. These examples are intended for paper-aligned structural analysis, annotation, comparison, and bounded refinement work only.
